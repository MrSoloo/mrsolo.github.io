Hexoの相关命令

本文参考部署 Hexo，主要是说一下指令，方便查看

🍕编译指令

  • 清除缓存 ,网页正常情况下可以忽略此条命令,执行该指令后,会删掉站点根目录下的public文件夹
hexo clean
  • 生成静态网页 (执行 $ hexo g后会在站点根目录下生成public文件夹, hexo会将”/blog/source/“ 下面的.md后缀的文件编译为.html后缀的文件,存放在”/blog/public/ “ 路径下)
hexo g

🍕本地部署

  • 启动本地服务器，用于预览主题。Hexo 会监视文件变动并自动更新，除修改站点配置文件外,无须重启服务器,直接刷新网页即可生效。 | 部署好可以在http://localhost:4000调试查看。
hexo s

🍕上传部署

  • 自动生成网站静态文件，并将本地数据部署到设定的仓库(如github)。上面的指令很多采用了简写，会发生冲突，尤其hexo deploy会和豆瓣部署冲突，所以建议这里不要简写。
hexo deploy # 简写为 hexo d

🍕安装插件

npm install 插件名 --save

🍕卸载插件

npm uninstall 插件名

🍕其他命令

  • 查看node.js版本号
node-v
  • 查看npm版本号
npm -v
  • 查看git版本号
git --version
  • 查看hexo版本号
hexo -v
  • 新建一篇博文
hexo n "我的第一篇文章"` 等价于 `hexo new "我的第一篇文章"` 还等价于 `hexo new post "我的第一篇文章"
